# 百晓册原生界面跳转（超链接式）

> 来源：反编译 `Assembly-CSharp`（`.ref/`，本地不入库）。本文是结论性笔记，不含反编译原文。

## 一句话

想让我们 IMGUI 面板里的"超链接"点一下就跳到游戏**原生百晓生**对应词条，**不需要把面板改成 UGUI**——借用游戏自己的窗口即可（与「实时人物编辑器」开化魂阁同一套反射借窗手法）。

## 官方入口

`Game.Views.Encyclopedia.ViewEncyclopediaPanel`：

```csharp
public static void OpenLink(EncyclopediaTipLinkItem link) {
    UIElement.Encyclopedia.SetOnInitArgs(EasyPool.Get<ArgumentBox>().SetObject("link", link));
    UIManager.Instance.ShowUI(UIElement.Encyclopedia, true);
}
// OnInit 里除了 "link" 还认一个更通用的字符串参数 "key"：
//   argsBox.Get("key", out key)  →  view.JumpKey = key  →  BasicInfoView.JumpTo(key)
```

我们走 **`"key"` 参数**这条更通用的路（任意词条都能跳，不限于预定义的 TipLink 枚举）：

```csharp
UIElement.Encyclopedia.SetOnInitArgs(new ArgumentBox().SetObject("key", key));
UIManager.Instance.ShowUI(UIElement.Encyclopedia, true);
```

## key 是什么（精确命中，无需退搜索）

`BasicInfoView.JumpTo(string key)`：

```csharp
NodeData node = EncyclopediaDataManager.Instance.GetNodeData(key);
if (node != null) JumpTo(node.Id, null);   // 精确命中 → 直达词条
else CalcSearchResult(key, ...);            // 对不上 → 退游戏自带搜索（不会开空窗）
```

而 `GetNodeData(string)` 查的是 `NodeKeyDict`，该字典正是用 **`EncyclopediaContentItem.Key`** 灌的：

```csharp
NodeKeyDict[config.Key] = node;   // config = EncyclopediaContent.Instance[index]
```

→ **传 `EncyclopediaContentItem.Key` 即精确命中**。我们 `EncyclopediaText.Render` 本来就在输出 `it.Key`，直接复用。容错也好：万一某 key 对不上，自动退到游戏搜索，不会开空窗。

## 实现位置（本项目）

- `Actions/EncyclopediaNavigator.OpenEntry(key, out error)`：纯反射开窗。`FindLoadedType("UIElement"/"UIManager"/"ArgumentBox")` → `UIElement.Encyclopedia` 静态成员 → `SetOnInitArgs(box)` → `UIManager.Instance.ShowUI(encyclopedia, true)`。
- `Actions/EncyclopediaQueryAction.RecentHits`：每次检索后记录命中的 `(TitlePath, Key)`，供 UI。
- `UI/ChatPanel.DrawEntryLinks()`：把命中渲染成可点按钮，点击调 `OpenEntry`。

## 注意

- **主线程 + 游戏内**：开窗须在 Unity 主线程、且已进入游戏（存档加载、百晓册可用）。在主菜单 `UIElement.Encyclopedia`/`UIManager.Instance` 可能为空，`OpenEntry` 会返回 false 而非崩溃。从 IMGUI 的 `OnGUI` 按钮里直接调用即可（已在主线程，RTMC 也是这么干的）。
- `RecentHits` 目前只保留**最近一次**检索的命中（一轮多次检索只留最后一次）；将来要"整轮累积"再改。

## 关联：RTMC 的另一处可复用资产

「实时人物编辑器」真正可复用的不是字体（见 [../ui/字体复用与天花板.md](../ui/字体复用与天花板.md)），而是两套手法：

1. **反射借原生窗口**（本文这套）：`UIManager` + `UIElement.X` + `ArgumentBox.SetOnInitArgs`；还能 `PatchNativeAvatarEditorTexts` 遍历 `TMP_Text` 改原生按钮文案。
2. **前后端跨域改游戏状态**：`ModDomainMethod.AsyncCall.CallModMethodWithParamAndRet(null, ModId, method, SerializableModData, 回调)`，参数/返回用 `SerializableModData`（Set/Get 键值袋），回调走 `AsyncMethodCallbackDelegate` + `RawDataPool` + `Serializer.Deserialize`。UI 域与游戏逻辑域分离，跨域改状态必须走这套序列化异步调用——将来加"让 agent 操纵游戏"的 action 时照此拆 Backend.dll。
