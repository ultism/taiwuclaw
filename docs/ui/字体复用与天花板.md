# IMGUI 字体复用与天花板

> 结论性笔记。记录"为什么我们的 IMGUI 面板做不到和游戏像素级一致的字体"，以及现状取舍。

## 结论（TL;DR）

- 游戏正文字体只以 **TMP SDF 资源**（`TMP_FontAsset`）存在，IMGUI（`GUIStyle.font` 只认 `UnityEngine.Font`）**碰不到**。
- 打包后 `TMP_FontAsset.sourceFontFile` 全为 **null**，拿不到源 TTF。
- 进程里可用的 `UnityEngine.Font` 只有系统通用字体（Arial/Consolas/LegacyRuntime），**没有游戏那款衬线 CJK**。
- 所以 IMGUI 这条路最多"蹭个像"：退到**系统衬线中文字体**（思源宋体/宋体）。要真正复用游戏字体图集，只有 UGUI+TMP 一条路——**风险太大，已放弃**。
- 想要游戏原生字体观感的地方，正解是**借用游戏自己的窗口**（见 [../反编译/百晓册界面跳转.md](../反编译/百晓册界面跳转.md)），而不是让 IMGUI 去复刻。

## 现场日志（实测）

```
已加载 TMP_FontAsset 8 个，全部 sourceFontFile=null：
  Font SDF BIG5 Core 1 / 2   （繁体 Big5）
  Font SDF GB2312            （简体，游戏正文就是这个）
  NotoSerifKR-SDF            （思源/Noto 衬线，武侠宋体味）
已加载 Font 3 个：
  LegacyRuntime、CONSOLA、arial   （全是系统通用字体）
```

## 一个澄清：「实时人物编辑器」也没复用字体

搜遍其反编译源，**没有任何 `.font` / `CreateDynamicFont` / `fontAsset`**。它自己的 IMGUI 面板用的就是 IMGUI 默认字体，和游戏字体无关。它"长得像游戏"只发生在它**调起游戏原生化魂阁 UGUI 窗口**时（那本就是游戏 TMP 文字）。换言之：它不是把游戏字体搬进面板，而是需要原生观感时直接借游戏窗口。

## 我们当前实现

`UI/GameFont.Resolve()` 四级回退：

1. `fontName`（llm.json 可指定，子串匹配）优先；
2. 反射扫 `TMP_FontAsset.sourceFontFile`（当前全 null，落空）；
3. 扫已加载 `Font`，排除系统通用字体黑名单（当前无游戏字体，落空）；
4. `Font.CreateDynamicFontFromOSFont(["Source Han Serif SC","思源宋体","SimSun","宋体",…], 16)` —— 退到系统衬线 CJK，蹭游戏宋体观感。

`UI/TaiwuStyles.EnsureInit()` 末尾把解析到的字体喂给全部 GUIStyle 的 `.font`。`llm.json` 的 `fontName` 字段可手动锁定（首启日志会列出所有候选字体名）。

## 若将来真要像素级一致

把聊天面板搬到 UGUI，直接用游戏的 `Font SDF GB2312` 这个 `TMP_FontAsset` 渲染。代价：重写 Canvas/TMP_InputField/ScrollRect/拖拽/缩放，需 UnityExplorer 侦察游戏 UI 层级。当前判定收益不抵风险，搁置。
